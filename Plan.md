# Popfolio — 구현 계획 (Plan.md)

> PRD를 바탕으로, "코드 짜기 전에" 합의한 구현 방향을 정리한 문서.
> 원칙(PRD와 동일): **욕심내지 않기.** 이번 목표는 "내 로컬에서 동작하는 데모"까지.
> 살아있는 문서 — 만들면서 계속 업데이트.

## 0. 이번에 합의한 것 (결정 완료)

| 항목 | 결정 |
|---|---|
| **생성 방식** | 예쁜 템플릿 + 자동 채우기 (AI 생성 아님) |
| **분위기(무드)** | 기본은 자동 제안(업로드 이미지 색에서 추출) → 사용자가 골라서 바꿀 수 있음 |
| **소개글 처리** | 사용자가 쓴 문구 그대로, 템플릿에 예쁘게 타이포그래피 배치 (AI 카피 없음) |
| **로그인/계정** | 없음 (MVP). 만들면 바로 링크 |
| **저장** | 서버에 파일로 저장 → `/p/[id]` URL로 다시 열기 & 공유 |
| **기술 스택** | Next.js (App Router) + Vercel 배포 지향 |
| **이번 목표** | 로컬에서 동작하는 데모 (실제 인터넷 배포는 다음 단계) |

## 1. 사용자 흐름 (Flow)

```
[랜딩 /]  →  [만들기 /create]  →  (자동으로 팝업 브랜드처럼 세움)  →  [내 페이지 /p/{id}]
                                                                         └→ 링크 복사해서 공유
```

1. **랜딩 `/`** — 한 줄 소개 + "내 팝업 브랜드 만들기" 버튼.
2. **만들기 `/create`** — 한 화면짜리 폼:
   - 브랜드 이름
   - 한 줄 소개 / 컨셉
   - 뭐 하는 사람인지 (짧은 소개)
   - 작업 아이템: 이미지 업로드 + 각 설명 + 순서(드래그 or 위/아래 버튼)
   - **분위기**: 첫 이미지 업로드 시 자동으로 무드 1개 제안 → 사용자가 다른 무드로 바꿀 수 있음
   - [세우기] 버튼 → 저장 후 `/p/{id}`로 이동
3. **내 페이지 `/p/{id}`** — 팝업 브랜드처럼 세워진 결과 페이지. 공유용 링크 복사 버튼.

## 2. "자동으로 팝업 브랜드처럼 세우기" — 심장 부분 설계

이 서비스의 핵심. 아래 3가지 조합으로 "알아서 근사하게" 느낌을 만든다.

### (a) 무드(Mood) 시스템
- 무드 = **팔레트 + 폰트 페어링 + 액센트 스타일** 묶음.
- MVP 무드 후보 (3~4개):
  - `modern` — 모던/시크 (뉴트럴 + 진한 액센트, 산세리프 볼드)
  - `warm` — 따뜻/포근 (크림·테라코타, 세리프 믹스)
  - `minimal` — 미니멀 (화이트 여백, 얇은 타이포)
  - `vivid` — 비비드/트렌디 (선명한 색 블록, 큰 타이포)
- **자동 제안**: 업로드된 대표 이미지에서 주요 색을 추출 → 가장 가까운 무드를 기본값으로.
- **사용자 변경**: 폼에서 무드 칩을 눌러 언제든 바꿀 수 있음. (PRD의 "분위기 골라서 다시 돌리기"와 연결)

### (b) 템플릿(레이아웃)
- MVP는 **레이아웃 1종 + 무드별 변주**로 시작 (욕심내지 않기).
- 페이지 구조(팝업스토어/브랜드 톤 — 미술관·이력서 아님):
  - **Hero**: 브랜드 이름(큰 타이포) + 한 줄 컨셉 + "뭐 하는 사람" 짧은 소개
  - **Works**: 작업 이미지 갤러리 (그리드/스택), 각 아이템 설명
  - **Footer**: 이름 재노출 + 공유(링크 복사) + "made with Popfolio"
- 색 블록·큰 여백·굵은 타이포로 "조용한 미술관"이 아니라 "트렌디한 브랜드" 느낌.

### (c) 카피(소개글)
- **입력 문구 그대로** 사용. AI 생성 없음.
- 서비스가 하는 일은 "쓰는 것"이 아니라 "**예쁘게 앉히는 것**" — 위계(hero/section), 폰트, 간격.

## 3. 데이터 모델 & 저장

로그인 없음 → 서버 **파일 저장**으로 공유 가능한 URL 확보.

```
data/
  portfolios/
    {id}/
      portfolio.json      # 아래 스키마
      images/
        {imageId}.{ext}   # 업로드 원본 (필요시 리사이즈본)
```

`portfolio.json` 스키마(초안):
```jsonc
{
  "id": "aB3xK9",                 // 짧은 고유 ID (nanoid)
  "createdAt": "2026-07-10T...",
  "brand": {
    "name": "브랜드 이름",
    "tagline": "한 줄 소개/컨셉",
    "about": "뭐 하는 사람인지"
  },
  "mood": "warm",                 // modern | warm | minimal | vivid
  "items": [
    { "id": "im_1", "image": "images/im_1.jpg", "caption": "설명", "order": 0 }
  ]
}
```

- **ID**: `nanoid` 짧은 문자열 → `/p/{id}`.
- 이미지: 업로드 시 서버에 저장. (데모에선 리사이즈 최적화는 선택)
- 나중에 확장: 이 파일 스키마 그대로 DB(예: SQLite/Postgres)로 옮기면 로그인·재편집 붙이기 쉬움.

## 4. 기술 스택 & 구조

- **프레임워크**: Next.js (App Router) + TypeScript
- **스타일**: Tailwind CSS (무드별 토큰을 CSS 변수/클래스로 관리)
- **이미지 색 추출**: `node-vibrant` 또는 `sharp` 기반 간단 추출
- **ID**: `nanoid`
- **저장**: 로컬 파일시스템 (`data/`) — 데모용. 배포 시엔 스토리지 교체 지점만 분리해 둠.

### 라우트 / API
| 경로 | 역할 |
|---|---|
| `/` | 랜딩 |
| `/create` | 만들기 폼 (클라이언트 상태로 아이템 관리) |
| `/p/[id]` | 결과 페이지 (서버 컴포넌트에서 `portfolio.json` 읽어 렌더) |
| `POST /api/portfolios` | 폼 제출 → 저장 → `{ id }` 반환 |
| `POST /api/upload` | 이미지 업로드 → 저장 경로 반환 |
| `POST /api/suggest-mood` | (선택) 이미지에서 무드 자동 제안 |

### 폴더 구조(초안)
```
app/
  page.tsx                # 랜딩
  create/page.tsx         # 만들기 폼
  p/[id]/page.tsx         # 결과 페이지
  api/portfolios/route.ts
  api/upload/route.ts
components/
  create/...              # 폼, 이미지 업로더, 무드 선택
  brand/...               # Hero, Works, Footer (무드 반응)
lib/
  storage.ts              # data/ 읽기·쓰기 (교체 지점 격리)
  moods.ts                # 무드 정의(팔레트/폰트)
  colorToMood.ts          # 이미지 색 → 무드 매핑
data/                     # 런타임 저장소 (gitignore)
```

## 5. 구현 단계 (Milestones)

- **M0. 프로젝트 셋업** — Next.js + TS + Tailwind, 기본 레이아웃, `data/` gitignore.
- **M1. 무드 시스템** — `moods.ts`(3~4개 팔레트/폰트), 무드별 브랜드 컴포넌트 스타일.
- **M2. 결과 페이지 `/p/[id]`** — 하드코딩 샘플 데이터로 Hero/Works/Footer 완성 (심장 먼저 눈으로 확인).
- **M3. 저장 계층** — `storage.ts`, `POST /api/portfolios`, 이미지 업로드.
- **M4. 만들기 폼 `/create`** — 입력 + 이미지 업로드 + 순서 + 무드 선택/자동 제안 → 제출 → 이동.
- **M5. 랜딩 `/` + 공유** — 링크 복사 버튼, 마무리.
- **M6. 로컬 데모 점검** — 실제로 "만들기 → 링크 → 다시 열기" 전체 흐름 확인.

> 순서 의도: **M2에서 "세워진 결과"를 먼저 눈으로 보고** 심장이 근사한지 확인한 뒤, 입력·저장을 붙임.

## 6. "이러면 완성" (이번 데모 기준 — PRD 5장)

로컬에서:
1. `/create`에서 작업 몇 개 + 몇 마디를 넣고
2. [세우기]를 누르면 팝업 브랜드처럼 근사한 `/p/{id}` 페이지가 뜨고
3. 그 URL을 다시 열어도 그대로 보이면 → **OK**.

(성공 신호: "어 이거 내가 만든 거 맞아? 나 뭐 하는 사람처럼 보인다" 반응)

## 7. 이번엔 안 하는 것 (다음으로)

- 로그인 / 내 포트폴리오 재편집 (저장 스키마는 확장 대비해 둠)
- 실제 사업 기능(문의·주문·결제)
- 실제 인터넷 배포(Vercel) — 데모 다음 단계
- AI 카피/AI 레이아웃 생성 — 지금은 템플릿+자동 채우기로 충분

## 8. 아직 열려 있는 작은 결정들 (구현하며 확정)

- 무드 최종 개수와 이름 (3개 vs 4개) — M1에서 눈으로 보고 확정
- 작업 아이템 순서 UI: 드래그 vs 위/아래 버튼 (간단한 버튼으로 시작 권장)
- 무드 자동 제안 정확도: 애매하면 그냥 기본 무드 하나로 시작하고 수동 선택 위주로
- 폰트: 웹폰트(무료) 페어링 선정 — 무드별 1쌍씩
```
